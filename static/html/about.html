About Jollybee Online Judge

What is "Jollybee"?

Jollybee is a group of competitive programming enthusiasts consisting of active and former students of Bina Nusantara University.

Jollybee Online Judge (JOJ)?

JOJ is being developed by Sandy Karunia – one of Jollybee member, and helped by William Liang. JOJ can be used for both formal and informal contests. JOJ focuses on adding problemset especially from Indonesian Contests (BNPCHS, INC, etc), but it is not impossible for problemset from other international contests to be added. Currently JOJ also hold some weekly contests called Jolly Challenge. Most of Indonesian problems are translated by Sandy Karunia and Ricky Chandra. You can ask any question (including discuss about a problem) in JOJ facebook fanpage.

Judge System

This online judge uses standard input and standard output as the I/O method. Standard error will also be considered in standard output stream.

Judge types:
  • Exact Match: The judge will compare user's output and jury's output without any tolerance. Different character count or line count will result in Wrong Answer verdict.
  • Ignore Multiple Whitespaces: The judge will compare user's output and jury's output with a tolerance of whitespace characters.
System specification:
  • Intel(R) Xeon(R) CPU E5-2630L v2 @ 2.40GHz
  • Ubuntu 14.04.3 LTS (GNU/Linux 3.13.0-52-generic x86_64)
Supported Languages:
  • Java 8
  • GNU C/C++/C++11 4.8.4
  • FPC 2.6.2-8
  • Python 2.7
  • Python 3.4
Available Verdicts:
  • Pending: Your submission is still in judge queue.
  • Accepted: Your program is correct.
  • Output Limit Exceeded: Your program produced too much output.
  • Time Limit Exceeded: Your program runs longer than specified time limit.
  • Wrong Answer: Your program produced incorrect output.
  • Compile Error: Your program can't be compiled.
  • Memory Limit Exceeded: Your program tried to use more memory than the specified memory limit.
  • Run Time Error: Your program crashed during execution.
  • Submission Error: There is something wrong in the server.
